## 评测框架：两个维度

### 维度一：Rubric 评分（考点评分）

将 PRD 中的要求拆解为具体的、可判断的考点。每个考点独立评分。

**考点设计原则**：

- 一个考点只考一件事
- 判断标准必须客观可描述，不能依赖"感觉好不好"
- 每个考点必须能回答：AI 回答了 X，如何判断这个考点是否通过？

**考点赋分（遵循 rubric-scoring/SKILL.md 第三节）**：

| 分段 | 分值 | 占比要求 |
|------|------|---------|
| 必答点 | 9–10 分 | **≥ 40%** |
| 重要点 | 6–8 分 | 20–50% |
| 附加点 | ≤ 5 分 | **≤ 15%** |
| 扣分点（负向） | -6 到 -10 分 | **≤ 20%** |

扣分点必须覆盖本 Prompt 场景中 AI 最可能犯的典型错误（模型幻觉、指令不遵循、行业通病）。

### 维度二：整体质量评分（General Quality）

从以下固定维度评估输出的整体质量，每项 1–5 分：

| 维度 | 说明 |
|------|------|
| 指令遵循度 | AI 是否完整执行了 Prompt 中的指令？ |
| 输出格式准确性 | 输出是否符合要求的格式？ |
| 内容质量 | 输出内容是否准确、有用、无幻觉？ |
| 一致性 | 相似输入是否产生一致的输出风格和质量？ |
| 约束遵守率 | AI 是否违反了任何行为约束？ |

> 注：整体质量评分的详细打分规则由 `skills/overall-scoring/SKILL.md` 定义，该文件优先级高于此处默认规则。如果 Skill 文件存在，以 Skill 文件为准。

## 维度一：Rubric 评分

### 一二级标题（思路骨架，不出现在评分表里）

> 根据 Prompt 类型，先构建评测骨架：
> 例：How-to 类 → 「用户分层」「方法模型」「风险边界」「验证信号」

### 考点列表

每条评分点写完后必须过 rubric-scoring/SKILL.md 的自检清单。

| ID | 类型 | 评分点（动作 + 具体事实） | 补充说明 | 赋分 |
|----|------|--------------------------|---------|------|
| R-01 | 得分点 | <具体动作 + 具体事实，不含指代词，不宽泛> | <边界说明、复合逻辑、示例（如有）> | 9 |
| R-02 | 得分点 | ... | ... | 7 |
| R-03 | 扣分点 | 出现了错误 A（描述具体错误） | <说明为何这是典型错误> | -8 |

**字段说明**：
- `类型`：得分点 / 扣分点
- `评分点`：核心考察内容，必须符合 P0-1 到 P0-6 规则
- `补充说明`：考点达成逻辑复杂时补充；简单考点可留空
- `赋分`：得分点 1–10，扣分点 -6 到 -10

### Rubric 总分计算

```
总得分 = Σ(得分点实际得分) + Σ(扣分点扣分)
满分 = Σ(所有得分点满分)
rubric_score = 总得分 / 满分
通过阈值（默认）：rubric_score ≥ 0.80 视为通过
```

---

## 维度二：整体质量评分

| 维度 | 权重 | 评分标准摘要 |
|------|------|------------|
| 指令遵循度 | 25% | 5=完全遵循，3=遗漏次要指令，1=忽略核心指令 |
| 输出格式准确性 | 20% | 5=格式完全正确，3=格式有小偏差，1=格式完全错误 |
| 内容质量 | 25% | 5=准确有用无幻觉，3=部分准确，1=大量错误或幻觉 |
| 一致性 | 15% | 5=相似输入输出高度一致，3=有波动但可接受，1=严重不一致 |
| 约束遵守率 | 15% | 5=无约束违反，3=违反次要约束，1=违反核心约束 |

### 整体质量总分计算

```
general_score = Σ(各维度得分 × 权重)
通过阈值（默认）：general_score ≥ 3.5 / 5 视为通过
```

---

## 综合判定规则

| 结果 | 条件 |
|------|------|
| PASS | rubric_score ≥ 0.80 且 general_score ≥ 3.5 |
| PARTIAL | 其中一项通过，另一项未通过 |
| FAIL | 两项均未通过 |

---

## 自动评分说明

Eval Runner 使用 AI Judge 模式进行自动评分。
- Rubric 评分：逐条判断，输出 0/1/2
- 整体质量评分：基于输出整体判断，输出 1–5

AI Judge 的 System Prompt 见 `skills/eval-runner/SKILL.md`。
